# Adoption Guide

Start with one workflow where reconstruction is already painful.

Good first candidates:

1. AI-assisted code changes,
2. CI validation handoffs,
3. release decisions,
4. human approvals with evidence,
5. failed automation runs that need later review.

Do not start by modeling every workflow. Emit Work Receipts for consequential
work where later reconstruction matters.

Success means a reviewer can answer:

1. what happened,
2. who or what acted,
3. what evidence existed,
4. what outcome was claimed,
5. what was accepted,
6. what should happen next.
